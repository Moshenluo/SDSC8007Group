# 结果真实性分析

## ⚠️ 潜在问题

### 1. 数据泄露风险

**问题**: 特征预处理（归一化）是在**整个数据集**上进行的，然后才划分训练/验证集

```python
# 错误做法（当前）
scaler.fit(all_data)  # 用了全部数据
train, val = split(all_data)

# 正确做法
train, val = split(all_data)
scaler.fit(train)  # 只用训练集
```

**影响**: 验证集信息泄露到训练集，导致性能**虚高**

---

### 2. ID 编码问题

**问题**: user_encoded 和 event_encoded 是在整个数据集上编码的

```python
# 验证集的用户 ID 在训练时已经见过
# 模型可能"记住"了某些用户的行为模式
```

**影响**: 对于**新用户**的泛化能力被高估

---

### 3. 类别不平衡

训练集标签分布：
- 感兴趣：26.8%
- 不感兴趣：3.3%
- 无操作：69.8%

**问题**: 模型可能学会了预测"大多数用户无操作"
